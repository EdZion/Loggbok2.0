# Post mortem
### Inledning 
Syftet med den här uppgiften var att göra en loggbok. De arbetssätt som användes var att man anväde sig utav en contoller, OOP och Mysql.
Det som skulle kunna utvecklas är att  göra så att de faktikst blir en loggbok som fungerar och även går att göra något med. 
### Bakgrund
Teknikera som används är OOP, Mysql, Controller och Swing UI designer. De här tekniker används för att de var ett krav för den här uppgiften.
OOP används för att det gör det lättare att hålla koll på vad man jobbar med. Controller används till att att koppla samman Swing UI Designer med OOP.
Swing UI designer används för att de är ett lätt sätt att få en design och en popup-fönster som ser okej ut.
MySql anväds som databas för att lagra datan som används till loggboken. 
### Positiva erfarenheter

Det gick väldigt bra att tänka ut vad de var jag skulle göra även om de var väldigt svårt att få de utskrivet så att de fungerar som kod.
### Negativa erfarenheter
De som inte gick så bra var att loggboken inte funger överhuvudtaget som den skulle. 
Det gick inte så bra att fördela tiden då jag fastnade med att gör OOP som man skulle göra i början av projektet.
De som jag känner att jag vill bli bättre på är att både fördela tiden bättre så att jag kan jobba med de andra delarna i projektet samt att blir bättre på att göra alla de delarna vi jobbade på då jag tyckte de var svårt att förstå i ett så stort projekt.

### Sammanfattning
Jag tyckte att jag inte lärde mig så mycket utav den här uppgiften då jag tyckte den var extremt svår att göra då jag inte är bra på att koda.
Men själva ideen utav vad jag skulle göra i tanken var faktisk helt okej även fast jag inte vet hur jag ska få det till kod.
Det finns mycket som skulle kunna bli bättre med det här projektet då det inte ens fungerar så skulle man kunna börja med att göra så att man faktisk kan skriva ner saker och även spara de som skrivs.
